# CLAUDE.md

Правила работы с кодом этого проекта (обязательны для всех изменений):

## Принципы исправления и проектирования

- Не лечи симптомы, не выявив корневую причину (root cause).
- Фикс делай на source-of-truth (owner layer), а не там, где симптом виден.
- Избегай компенсации на дочернем слое: fallback'ов, заплаток, дублированной логики, ветвлений-костылей.
- Перед фиксом всегда выполняй ultra-deep систему исследование end-to-end:
  - top-down: route → page → container → orchestration → state
  - bottom-up: function → hook → service → API → DB
- Диагностика по слоям: data/contracts → business logic → async/timing → UI state → integration → architecture.
- Если баг появился в дочернем слое — сперва инспектируй родительский / владельческий слой.
- Меняя механику — выравнивай все непосредственно связанные слои: contracts, handlers, queries, cache, serializers, loading/error states.
- Будь скептичен к фиксам в одном файле; обоснуй, почему другие слои не затронуты.
- Для frontend-проблем (WinForms / UI) инспектируй полный поток: форма → контроллер/презентер → сервис → репозиторий → БД.
- Предпочитай системные фиксы, но соблюдай пропорциональность изменений. Если требуется ре-архитектура — определи scope, risks, compatibility и rollout order.
